To keep our code organised we keep utilities such as root Queries and Commands in separate files. Utilities that require context from a ModelItem (Activity, Component) should be put in the commands/queries section of those components.

The following items are considered utilities:

  • ODT Commands/Queries in the file root
  • OMT Procedure
  • OMT Standalone Query
The following items are considered model items that should populate without utilities:
  • OMT Activity
  • OMT Component
  • OMT Ontology
  • OMT Loadable

Examples

The following code snippet would PASS:

commands: |
    DEFINE COMMAND myCommand => {}

queries: |
    DEFINE COMMAND queries => {}

model:
    MyProcedure: !Procedure
        onRun:

    MyStandaloneQuery: !StandaloneQuery
        query:

the following code snippet would FAIL:

commands: |
    DEFINE COMMAND myCommand => {}

queries: |
    DEFINE COMMAND queries => {}

model:
    MyActivity: !Activity
        onStart:

    MyStandaloneQuery: !StandaloneQuery
        query:
