If an import member is declared but not used, it is dead code and should be removed.
You can exclude certain files from this analysis where it should be explicitly permitted to import members that are not used within its own scope. For example, the index.omt used by modules to expose functionalities to other modules.

Parameters

exclusion
Defines one or more filenames (; seperated) that are allowed to import members

Examples

With exclusion = index.omt the following index.omt code snippet would PASS:

import:
    myFile:
    - importedMember

The following regular-file.omt code snippet would PASS:

import:
    myFile:
    - importedMember

model:
    MyActivity: !Activity
        onStart: |
            @LOG(importedMember)
